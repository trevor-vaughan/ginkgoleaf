# Security Policy

## Supported Versions

ginkgoleaf is pre-1.0. Only the latest tagged release receives security fixes.

## Reporting a Vulnerability

Please report security issues privately via GitHub's
[private vulnerability reporting](https://github.com/trevor-vaughan/ginkgoleaf/security/advisories/new).
Do not open a public issue for security problems.

You can expect an initial acknowledgement within 7 days. Once a fix is ready,
we will coordinate a release and credit reporters who wish to be named.

## Verifying Releases

Every release ships SBOMs, cosign signatures, and SLSA build provenance.
See the "Verifying releases" section of the README for verification steps.
